校园答疑智能客服 AI Agent

开发文档 - 完整技术指南

一、项目概述

1.1 项目目标

构建一个基于大语言模型（LLM）的校园答疑智能客服系统，能够回答学生关于校园生活、教务管理、选课、考试、图书馆、食堂、宿舍等方面的常见问题。通过API调用云端大模型，无需本地GPU硬件。

1.2 核心功能（已实现）

  • 智能问答：基于校园知识库（7大类FAQ，100+问答对）回答学生问题
  • 多轮对话：支持上下文理解，连续对话，会话历史持久化到JSON文件
  • RAG检索增强：ChromaDB向量检索 + BGE-small-zh Embedding，精准匹配相关知识
  • 流式输出：SSE流式返回，实时显示回答生成过程
  • 自定义Web前端：类ChatGPT风格界面，支持会话管理、快捷问题、消息操作
  • 模型缓存：Embedding模型单例缓存 + 检索结果LRU缓存，提升响应速度

1.3 技术特色

  • 零GPU要求：通过小米Mimo API调用云端大模型
  • 低门槛：适合个人开发者或小团队
  • 可扩展：知识库可灵活更新，支持多格式文档
  • 高性能：模型预加载 + 检索缓存，首次响应快速

六、关键技术详解

6.1 RAG（检索增强生成）原理

用户提问 → Embedding → 向量检索 → Top-K 文档片段 ↓ Prompt 组装 ↓ LLM 生成回答 → 返回用户

6.2 文本分块策略

  • 按问答对分块：每对 Q&A 作为一个 chunk（推荐）
  • 按段落分块：每 200-500 字为一个 chunk
  • 重叠窗口：相邻 chunk 重叠 50-100 字，避免信息断裂

6.3 Prompt 工程

关键要素：

  1. 角色设定：明确 AI 助手身份
  2. 参考资料：注入检索到的知识片段
  3. 回答约束：要求基于参考资料回答，不编造
  4. 格式要求：简洁友好的语言风格

七、常见问题与解决

问题 解决方案
模型回答不准确 优化Prompt模板，增加检索数量，检查知识库内容是否覆盖该问题
首次响应慢 Embedding模型预加载（启动时自动完成），后续请求走缓存
中文乱码 确保文件编码为 UTF-8，环境变量设置 HF_ENDPOINT 镜像
API 调用失败 检查 config/.env 中的 XIAOMI_API_KEY 是否正确，网络是否通畅
模型输出包含思考过程 已通过 no_think 参数和 _parse_answer 方法过滤思考过程
知识库未更新 修改FAQ文件后，重新运行 python src/core/knowledge_loader.py 构建
