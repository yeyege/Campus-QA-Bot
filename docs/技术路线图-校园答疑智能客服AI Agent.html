前端：自定义HTML/CSS/JS（类ChatGPT风格）+ Gradio备用 后端：FastAPI（高性能异步API） 大模型：小米 Mimo API（mimo-v2.5，在线调用） 向量数据库：ChromaDB（轻量，嵌入式部署） Embedding：BGE-small-zh-v1.5（轻量中文语义理解，启动快） 缓存：模型单例缓存 + 检索LRU缓存

五、技术原理简述

5.1 什么是 RAG？

传统 LLM： 问题 → LLM → 回答（可能编造） RAG： 问题 → 检索相关文档 → 文档+问题 → LLM → 回答（基于事实）

RAG 的好处：

  • 回答更准确（基于实际文档）
  • 可随时更新知识库
  • 减少模型幻觉（胡说八道）

5.2 Embedding 是什么？

"选课系统在哪里？" → [0.12, 0.85, 0.33, ...] （一串数字） 通过向量相似度，找到最相关的文档片段

5.3 向量数据库是什么？

传统数据库：WHERE name = '张三'（精确匹配）

向量数据库：找到与问题意思最接近的 5 段文字（语义匹配）
