# 校园答疑智能客服 - 项目优化方案

---

## 一、项目目标

### 1.1 项目愿景
构建一个高效、准确、易用的校园答疑智能客服系统，帮助学生快速获取校园生活、教务管理、选课、考试、图书馆、食堂、宿舍等方面的常见问题解答。

### 1.2 核心目标

| 目标 | 描述 | 衡量指标 | 状态 |
|------|------|----------|------|
| **准确性** | 回答基于知识库，准确率高 | 回答准确率 ≥ 85% | ✅ 已实现 |
| **响应速度** | 快速响应学生问题 | 首次响应 ≤ 2秒，流式输出 | ✅ 已实现 |
| **易用性** | 界面友好，操作简单 | 用户无需培训即可使用 | ✅ 已实现 |
| **可扩展** | 知识库可灵活更新 | 支持多格式文档导入 | ✅ 已实现 |
| **稳定性** | 7x24小时稳定运行 | 可用性 ≥ 99% | ⚠️ 待验证 |

### 1.3 用户场景
- **学生**：查询选课、成绩、食堂、宿舍等常见问题
- **新生**：了解校园设施、报到流程、入学指南
- **访客**：查询校园位置、联系方式、开放时间

## 二、已完成优化项

### 2.1 性能优化 ✅

| 优化项 | 方案 | 实现文件 | 效果 |
|--------|------|----------|------|
| 模型预加载 | 启动时预加载Embedding模型，全局复用 | `src/core/cache.py` | 冷启动后响应提速 |
| 模型单例缓存 | ModelCache单例模式，避免重复加载 | `src/core/cache.py` | 内存占用降低 |
| 检索结果缓存 | LRU缓存常见问题检索结果 | `src/core/cache.py` | 重复问题秒级响应 |
| HuggingFace镜像 | 使用国内镜像加速模型下载 | 多处配置 | 下载速度提升 |

### 2.2 功能完善 ✅

| 功能 | 描述 | 实现文件 | 状态 |
|------|------|----------|------|
| 多轮对话 | 支持上下文理解，最近4轮历史 | `src/core/agent.py` | ✅ 完成 |
| 会话历史持久化 | 对话记录保存到JSON文件 | `src/core/conversation.py` | ✅ 完成 |
| 流式输出 | SSE流式返回，实时显示生成过程 | `src/api/chat.py` + 前端 | ✅ 完成 |
| 会话管理 | 新建/切换/删除会话 | 前端 `index.html` | ✅ 完成 |
| 知识库扩充 | 从4个分类扩展到7个（桂林理工特色） | `knowledge_base/faq/` | ✅ 完成 |
| 自定义前端 | 类ChatGPT风格界面 | `src/frontend/index.html` | ✅ 完成 |
| 模型思考过滤 | 过滤Mimo模型的思考过程输出 | `src/core/llm.py` + `agent.py` | ✅ 完成 |

### 2.3 架构优化 ✅

| 优化项 | 描述 | 实现文件 |
|--------|------|----------|
| API路由模块化 | 聊天API独立为chat.py路由 | `src/api/chat.py` |
| Agent单例模式 | 全局Agent实例，懒加载 | `src/api/chat.py` |
| 错误处理 | API异常统一捕获返回 | `src/api/chat.py` |
| 配置分离 | .env管理API密钥，config.yaml管理参数 | `config/` |

## 三、当前仍存在的问题

### 3.1 功能缺失

| 问题 | 描述 | 优先级 | 建议方案 |
|------|------|--------|----------|
| 无用户反馈 | 无法收集回答满意度 | 中 | 添加点赞/点踩按钮，存储到JSON |
| 无意图识别 | 所有问题走同一检索流程 | 低 | 简单关键词分类路由 |
| 无单元测试 | 核心模块无测试覆盖 | 中 | 使用pytest编写测试 |
| 无日志系统 | print输出，无结构化日志 | 低 | 使用Python logging模块 |

### 3.2 架构待改进

| 问题 | 描述 | 优先级 | 建议方案 |
|------|------|--------|----------|
| 无Docker配置 | 缺少容器化部署 | 低 | 编写Dockerfile + docker-compose.yml |
| 无CI/CD | 缺少自动化测试和部署 | 低 | GitHub Actions |
| 前端会话持久化 | 刷新页面后会话不丢失 | 中 | ✅ 已完成 (localStorage) |
| 无类型注解 | 部分代码缺少类型提示 | 低 | 逐步添加 |

### 3.3 性能待优化

| 问题 | 描述 | 优先级 | 建议方案 |
|------|------|--------|----------|
| 并发能力 | 单进程，无异步优化 | 中 | 使用asyncio异步化 |
| 冷启动时间 | Embedding模型加载需5-10秒 | 低 | 已通过预加载缓解 |
| 检索阈值 | 相似度阈值固定，不够灵活 | 低 | 动态调整阈值 |
