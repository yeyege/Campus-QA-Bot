
## 五、技术原理简述

### 5.1 什么是 RAG？

```
传统 LLM：  问题 → LLM → 回答（可能编造）

RAG：       问题 → 检索相关文档 → 文档+问题 → LLM → 回答（基于事实）
```

RAG 的好处：
- 回答更准确（基于实际文档）
- 可随时更新知识库
- 减少模型幻觉（胡说八道）

### 5.2 Embedding 是什么？

```
"选课系统在哪里？" → [0.12, 0.85, 0.33, ...] （一串数字）

通过向量相似度，找到最相关的文档片段
```

### 5.3 向量数据库是什么？

传统数据库：`WHERE name = '张三'`（精确匹配）
向量数据库：`找到与问题意思最接近的 5 段文字`（语义匹配）

## 六、新手常见疑问

### Q1：我没有 GPU 能做吗？
**可以！** 本项目使用 API 调用方式，用 DeepSeek API，普通电脑就能跑。

### Q2：需要多少 Python 基础？
会基本的变量、函数、列表操作即可。本项目代码量约 500-800 行，有注释讲解。

### Q3：需要多少时间？
每天投入 3-4 小时，约 10 天可以完成基础版本。

### Q4：知识库需要多少内容？
初始版本准备 50-100 个常见问题即可，后续可逐步扩充。

### Q5：部署到云服务器需要多少钱？
轻量应用服务器约 50-100 元/月，够用于学习和小规模使用。
